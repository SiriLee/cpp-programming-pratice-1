# 实验一：AI编码与人工编码的代码质量对比

本实验通过使用AI编程工具生成代码以及对静态代码进行分析，客观评估AI生成代码的优缺点，了解如何安全、高效地使用AI辅助编程，并培养代码审查与重构能力。

## 一、实验目的

1. 掌握使用AI编程工具生成、审查、改进代码的规范流程。
2. 学会使用静态代码分析工具（如Cppcheck，SonarQube等）评估C++代码质量。
3. 培养批判性思维：能够识别并修正AI生成代码中的错误、低效或不安全部分。
4. 产出一份完整的代码质量对比分析报告，为后续团队开发或个人项目提供参考。

## 二、实验任务

在EOJ平台的课程习题集中，自行选择一道“自己不会做、做起来有困难，或认为有一定挑战性”的题目，分阶段完成以下任务。

### 1. 阶段A：无AI辅助，纯手工实现C++代码

**步骤：**

1. 阅读并理解题目要求。
2. 使用AI编程工具对题目进行发散性讨论，但提示词明确说明不要给代码，可以讨论如下话题（但不局限）：
   - 这道题可能用什么算法？
   - 有哪些边界情况？
   - 暴力方法和优化方法分别是什么？
   - 时间复杂度和空间复杂度如何分析？
3. 在AI提供思路之后，独立编写一版自己手写的C++代码。
4. 运行测试，提交EOJ平台，记录测试结果。

### 2. 阶段B：使用AI编程工具生成代码

**步骤：**

1. 让AI根据相同的思路，生成一版完整C++代码。不要修改AI生成的代码。
2. 运行测试，提交EOJ平台，记录测试结果。

### 3. 阶段C：代码审查和质量改进

1. **人工审查代码**：逐行审查【可借助AI工具，向其询问审查范围，不可直接让其识别代码错误，保存对话记录】，例如：
   - 是否存在逻辑错误（如递归终止条件错误、类型转换错误）？
   - 是否处理了所有边界条件（如空字符串、空白字符）？
   - 是否有安全风险？
   - 变量命名是否清晰？函数是否过长？
2. 运行静态代码分析工具Cppcheck（或者其它工具）分别分析两版代码的代码质量。
3. 根据Cppcheck结果、个人测试结果和EOJ提交结果，对两版代码分别进行修改和完善。

### 4. 阶段D：整理完整实验过程，撰写实验报告。

## 三、实验要求

### 1. 题目选择要求

1. 可以来自EOJ课程习题集或者平时练习遇到的题目。
2. 不建议选择过于简单的入门题，例如只涉及单个公式计算、简单输入输出的题目。
3. 所选题目应能体现一定的算法思考过程，例如枚举、排序、贪心、动态规划、搜索、数据结构、字符串处理等。

### 2. AI使用要求

1. 在阶段A，阶段B，阶段C过程，必须保留与AI的主要对话记录。
2. 对话记录应能体现你如何提问、如何追问、如何判断AI回答是否可靠。
3. 不允许只提交“请直接生成代码”的单轮对话。

### 3. 代码要求

1. 至少提交四份代码：自己初始版本代码；AI初始生成代码；自己版本修改后代码；AI版本修改后代码。
2. 修改后代码应能正常编译，鼓励至少有一份最终代码通过EOJ测评，代码语言为C++。

### 4. Cppcheck使用要求

1. 分别对自己初始代码和AI初始代码运行Cppcheck。
2. 报告中需要保留Cppcheck的主要输出结果。
3. 需要解释Cppcheck报告中的问题，例如：变量未初始化；数组越界风险；无效条件判断；冗余代码；性能低效；风格问题等。
4. 如果Cppcheck没有发现问题，也需要说明是否代表代码一定正确，并结合测试或人工分析说明原因。

## 四、实验报告要求

实验报告需要包含以下内容：

1. **题目信息**：EOJ题号【或其他来源】；题目名称；题目简要描述；选择该题的原因。
2. **解题思路分析**：自己最初的理解；AI给出的主要思路；对不同思路的比较；最终采用的算法；时间复杂度和空间复杂度分析。
3. **AI对话过程记录**：至少保留关键的3至5轮对话；说明哪些回答有帮助；说明哪些回答存在错误、遗漏或不够严谨之处。
4. **两版初始代码对比**：自己初始代码的特点；AI初始代码的特点；两者在算法、边界处理、代码风格上的差异。
5. **Cppcheck分析结果**：自己代码的Cppcheck输出；AI代码的Cppcheck输出；对主要问题的解释；哪些问题是真问题，哪些可能是误报。
6. **修改与完善过程**：自己代码修改了哪些地方；AI代码修改了哪些地方；修改前后是否通过更多测试；是否通过EOJ，若未通过，需要分析原因。
7. **实验反思**：AI在本次实验中真正帮到了什么；AI生成代码中暴露了哪些问题；今后如何更可靠地使用AI辅助编程。

## 五、提交文档资料

1. 所有文档压缩为一个压缩包，命名格式建议为：`学号_姓名_代码质量分析实验.zip`
2. 压缩包中应包含：实验报告（word或者PDF版本）、四份完整的代码，建议命名为：
   - `self_initial.cpp`
   - `ai_initial.cpp`
   - `self_revised.cpp`
   - `ai_revised.cpp`
